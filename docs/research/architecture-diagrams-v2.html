Memory subsystems with v2 hierarchical organization. Episodic memory nests events into a four-level chunk hierarchy (event → episode → schema → domain). 04Retrieval Data Flow v2CUE-BASED + MULTI-DIMENSIONAL

The retrieval layer's v2 data flow. The Query Router fans out across four retrieval modes — temporal, cue-based, hierarchical, and semantic — to multiple subsystems in parallel. Results are ranked (including cue match score and hierarchical granularity), annotated with provenance (including taxonomy version), and composed within budget.

05Dynamic Taxonomy Lifecycle4-PHASE EVOLUTION

The taxonomy evolution pipeline manages the lifecycle of new concepts. Four phases — Discovery, Proposal (with concept formation), Validation, and Merge — handle unrecognized concepts end-to-end. Semantic drift detection continuously monitors existing concepts at concept and structural levels. All changes are tracked with full provenance.

06Storage Tiers v2HIERARCHICAL ORGANIZATION

Five storage tiers (T1–T5) with v2 hierarchical annotations. T2 stores chunk hierarchy and dual-index. T4 stores SKOS broader/narrower as typed graph edges. T5 archives historical taxonomy versions for provenance queries. The multi-tier model follows MemGPT's OS-style paging.

07End-to-End Agent Cycle v2EVOLUTION FEEDBACK LOOPS

A single agent cycle through the v2 architecture. The cycle adds three v2 feedback loops: (1) chunk hierarchy assignment during event recording, (2) consolidation into the taxonomy, and (3) as-needed taxonomy evolution when unrecognized concepts are encountered. All loops feed back into the architecture, making it self-improving.
